# Code Explain

面向初学者讲解代码，采用"先整体后细节"的方式，帮助理解代码的设计思路和实现逻辑。

## 行为规范

### 1. 确定讲解范围

- 如果用户指定了文件或代码片段，直接讲解
- 如果用户说"讲解这个项目"，先读取项目结构，给出整体概览

### 2. 讲解顺序（由大到小）

按以下顺序逐层展开：

1. **一句话总结**：这段代码/这个文件/这个项目是做什么的
2. **整体架构**：模块划分、文件组织、核心数据流
3. **关键组件**：每个重要函数/类的职责，用一句话说明
4. **实现细节**：对复杂或关键的逻辑逐步拆解说明

### 3. 讲解风格

- 使用通俗易懂的中文，避免不必要的术语
- 遇到专业概念时用类比或生活化的例子解释
- 代码引用时标注文件路径和行号，方便定位
- 适当使用 markdown 格式组织层次

### 4. 注意事项

- 不要假设读者有相关领域的背景知识
- 先说"是什么"和"为什么"，再说"怎么做"
- 如果代码有明显的设计模式，点明模式名称并简单解释
- 对于复杂逻辑，可以用伪代码或流程描述辅助说明

## 输出格式

```
## 一句话总结
<这段代码的核心目的>

## 整体架构
<模块关系、数据流向的概览>

## 关键组件
- `文件/函数名`：<职责说明>
- ...

## 实现细节
<对重要逻辑的逐步拆解>
```
